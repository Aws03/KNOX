# CLAUDE.md

Permanent context for Claude Code sessions working on **KNOX** (JadaraITKnowledgeSystem). Read this before making changes.

## 1. Project Overview

KNOX is a university knowledge hub (LMS) for IT students. Students belong to a **University → Faculty → Major** hierarchy, enroll in **Courses**, browse **course materials** organized in hierarchical folders, and take **quizzes** — either hand-authored by users with the "Writer" role, or auto-generated by AI from uploaded course materials.

## 2. Tech Stack

- **Backend**: .NET 10, Clean Architecture (4 projects: Domain / Application / Infrastructure / API), EF Core 10 + SQL Server, ASP.NET Identity + JWT, MediatR (CQRS), FluentValidation, AutoMapper, Swagger.
- **Frontend**: `../KNOX_Frontend/uni-hub` — React 19 + TypeScript + Vite, shadcn/ui + Tailwind v4, React Router v7, TanStack Query, Axios.
- **Database**: SQL Server.
- **Email**: Brevo (primary), AhaSend (fallback if `Brevo:ApiKey` isn't configured).
- **File storage**: local disk (`wwwroot/uploads`), served via ASP.NET static files.
- **Containerization**: Docker Compose (backend + frontend containers; connects to an existing `sqlserver` container via `host.docker.internal`, not a container defined in this repo's compose file).

## 4. Architecture Rules — Strict, Never Break These

1. **Domain has zero external dependencies.** No EF Core, no ASP.NET, no MediatR types beyond the `INotification` marker already in use for domain events. Domain entities use private setters and static `Create()` factories returning `Result<T>`.
2. **Application depends only on Domain.** It defines interfaces (ports) for everything it needs from the outside world (`IApplicationDbContext`, `IStorageService`, `IEmailService`, etc.) — it never references Infrastructure or API.
3. **Infrastructure implements Application's interfaces.** All EF Core, external HTTP calls (Brevo, OpenAI), identity, and file I/O live here.
4. **API is the composition root.** All DI wiring, middleware, and Kestrel/auth configuration lives in `Program.cs`. Controllers only translate HTTP ↔ MediatR.
5. **Never add business logic to Controllers.** A controller action should: build a command/query, `await _mediator.Send(...)`, map the `Result<T>` to an HTTP response. Validation, authorization *logic* (not the `[Authorize]` attribute itself), and business rules belong in the Application layer's handlers/validators or the Domain entities — not in a controller.

## 5. Key Patterns

- **CQRS via MediatR**: every use case is a `Command` or `Query` record + a `Handler` in `Application/Features/<Context>/{Commands,Queries}/<UseCase>/`. Controllers only send these through `IMediator`.
- **`Result<T>` pattern** (`Domain/Common/Results`): domain and application operations return `Result<T>` instead of throwing for expected failures. Check `.IsSuccess`/`.IsError`, use `.Match(onValue, onError)` or read `.Value`/`.Errors`. `Result<T>.Value` returns `default!` (not an exception) when `IsError` — always check success first.
- **FluentValidation pipeline**: validators are auto-discovered per command and run by `ValidationBehavior` before the handler executes; a failing validator short-circuits to a `Result` error, the handler body never runs.
- **`TransactionBehavior`**: wraps any request whose type name ends in `Command` in an EF Core execution-strategy transaction automatically — handlers don't manage transactions themselves.

## 6. Running Locally (no Docker)

Backend:
```bash
cd JadaraITKnowledgeSystem.API
dotnet user-secrets list          # verify secrets are set (JWT, OpenAI, Brevo/AhaSend, SQL connection string)
dotnet run                        # listens on http://localhost:5001 (+ https://localhost:6001 outside containers)
```
Secrets (JWT secret, OpenAI key, Brevo/AhaSend keys, SQL connection string) live in `dotnet user-secrets`, **not** `appsettings.json` — that file only holds placeholders (`REPLACE_WITH_USER_SECRET`). See `appsettings.json.example` for the full shape.

Frontend:
```bash
cd ../KNOX_Frontend/uni-hub
npm run dev                       # http://localhost:5173, calls the API at http://localhost:5001/api by default
```

## 7. Running with Docker

```bash
cd ..            # the SP/ folder, parent of KNOX_Backend and KNOX_Frontend
cp .env.example .env    # fill in real secrets
docker compose up --build -d
```
- Backend container: `knox-backend`, port 5001, healthcheck-gated (bash `/dev/tcp` check — the aspnet runtime image has no curl/wget).
- Frontend container: `knox-frontend`, nginx on port 5173→80, proxies `/api/` and `/uploads/` to `http://knox-backend:5001`; `depends_on: backend: condition: service_healthy`.
- Connects to the **existing** `sqlserver` container via `host.docker.internal` — this repo's compose file does not define its own database service.
- `VITE_API_URL` is a **build-time** arg (Vite inlines it), defaults to `/api` in Docker so the browser never needs the backend's address directly.

## 9. Known Issues / Technical Debt

- **`IIdentityUserService.CreateAsync`'s `Result.IsSuccess` is always `true`.** Both its success and failure branches return via the value-tuple implicit conversion rather than the error conversion, so `Result.IsSuccess` never reflects an actual Identity failure. The real signal is inside the tuple: `identityUserId == 0` or a non-empty `errors` collection. `CreateUserCommandHandler`'s existing `if (!identityCreate.IsSuccess)` check does **not** catch this — it's a live bug in the registration flow. `DataSeeder` works around it by checking the tuple directly; the shared service itself has not been fixed (out of scope so far — touching it affects the live registration endpoint).
- **No automated tests exist anywhere** in the backend or frontend (no xUnit/NUnit/MSTest, no Jest/Vitest/RTL). No CI pipeline either (`.github/workflows/` is empty).
- **AutoMapper 12.0.1 has a known high-severity vulnerability** (`GHSA-rvv3-g6hj-g44x`) — flagged repeatedly in `dotnet build`/`docker build` output. Needs a version upgrade; not yet done.
- AI quiz generation's text extraction (`TextExtractionService`) is still a placeholder — `itext7`/`DocumentFormat.OpenXml` are installed but the actual PDF/DOCX/PPTX extraction methods return hardcoded placeholder strings, not real extracted text. This is Phase 4 below.
- A generated migration (`RenameIsVerifiedColumns`) and the current `AppDbContextModelSnapshot.cs` are entangled with an in-progress, uncommitted AI-quiz-generation feature in the same snapshot file — check `git status` before assuming the migration history is fully committed.
