# Quiz Generation Feature Migration Guide

## Overview
This migration adds support for AI-powered quiz generation from course materials.

## Changes

### New Tables

#### 1. SystemSettings
- `Id` (int, PK)
- `Key` (nvarchar(100), unique index)
- `Value` (nvarchar(500))
- `Description` (nvarchar(200), nullable)
- `IsEnabled` (bit, default: true)
- `CreatedAt` (datetime2)
- `UpdatedAt` (datetime2, nullable)
- `CreatedBy` (nvarchar(100), nullable)
- `UpdatedBy` (nvarchar(100), nullable)

**Seed Data:**
- Features.QuizGeneration.Enabled = "true"
- Features.QuizGeneration.MaxConcurrent = "3"
- Features.QuizGeneration.DefaultQuestions = "8"

#### 2. QuizGenerationJobs
- `Id` (int, PK)
- `MaterialId` (int, FK to CourseMaterials)
- `CourseId` (int, FK to Courses)
- `RequestedByUserId` (int, FK to Users)
- `Status` (int) - enum: Pending=0, Extracting=1, GeneratingQuizzes=2, Completed=3, Failed=4, Cancelled=5
- `ErrorMessage` (nvarchar(1000), nullable)
- `GeneratedQuizCount` (int, default: 0)
- `GeneratedQuizIds` (nvarchar(500), nullable) - JSON array
- `Options` (nvarchar(1000)) - JSON object
- `CompletedAt` (datetime2, nullable)
- `CreatedAt` (datetime2)
- `UpdatedAt` (datetime2, nullable)
- `CreatedBy` (nvarchar(100), nullable)
- `UpdatedBy` (nvarchar(100), nullable)

**Indexes:**
- IX_QuizGenerationJobs_MaterialId
- IX_QuizGenerationJobs_Status
- IX_QuizGenerationJobs_CreatedAt

**Foreign Keys:**
- MaterialId ? CourseMaterials.Id (ON DELETE CASCADE)
- CourseId ? Courses.Id (ON DELETE NO ACTION)
- RequestedByUserId ? Users.Id (ON DELETE NO ACTION)

### Modified Tables

#### Quizzes
**New Columns:**
- `Source` (int, default: 0) - enum: Manual=0, AIGenerated=1
- `SourceMaterialId` (int, nullable, FK to CourseMaterials)
- `PartNumber` (int, nullable)
- `TotalParts` (int, nullable)

**New Indexes:**
- IX_Quizzes_SourceMaterialId
- IX_Quizzes_Source

**New Foreign Keys:**
- SourceMaterialId ? CourseMaterials.Id (ON DELETE SET NULL)

#### CourseMaterials
**New Columns:**
- `HasGeneratedQuizzes` (bit, default: false)
- `AutoGeneratedQuizCount` (int, default: 0)

**New Indexes:**
- IX_CourseMaterials_HasGeneratedQuizzes

## Migration Commands

### 1. Generate Migration
```bash
cd JadaraITKnowledgeSystem.Infrastructure
dotnet ef migrations add AddQuizGenerationFeature --startup-project ../JadaraITKnowledgeSystem.API
```

### 2. Update Database
```bash
dotnet ef database update --startup-project ../JadaraITKnowledgeSystem.API
```

### 3. Verify Migration
```sql
-- Check SystemSettings table
SELECT * FROM SystemSettings;

-- Check QuizGenerationJobs table
SELECT COUNT(*) FROM QuizGenerationJobs;

-- Check new Quiz columns
SELECT TOP 1 Id, Source, SourceMaterialId, PartNumber, TotalParts FROM Quizzes;

-- Check new CourseMaterial columns
SELECT TOP 1 Id, HasGeneratedQuizzes, AutoGeneratedQuizCount FROM CourseMaterials;
```

## Rollback Instructions

If you need to rollback this migration:

```bash
cd JadaraITKnowledgeSystem.Infrastructure
dotnet ef database update <PreviousMigrationName> --startup-project ../JadaraITKnowledgeSystem.API
dotnet ef migrations remove --startup-project ../JadaraITKnowledgeSystem.API
```

## Post-Migration Steps

1. **Verify seed data:**
   ```sql
   SELECT * FROM SystemSettings WHERE [Key] LIKE 'Features.QuizGeneration%';
   ```

2. **Test feature flag:**
   - Call the API endpoint to check feature flags
   - Toggle quiz generation feature on/off

3. **Test quiz generation:**
   - Upload a PDF/DOCX/PPTX material with `GenerateQuiz=true`
   - Check QuizGenerationJobs table for job creation
   - Monitor job status until completion
   - Verify generated quizzes in Quizzes table

## Notes

- The migration includes default values for backward compatibility
- Existing quizzes will have `Source = Manual` by default
- Existing materials will have `HasGeneratedQuizzes = false` by default
- The feature can be disabled via SystemSettings without data loss
